# NOT-visionOS

웹캠 기반 손 제스처 입력으로 브라우저 안에서 다중 창을 조작하는 **visionOS 유사 웹 작업대** 프로젝트입니다.

## 프로젝트 한 줄 정의

> 웹캠으로 손가락 방향과 손모양을 읽어, 브라우저 안의 여러 창을 손으로 집고 움직이는 웹 작업대

## 목표 범위

이 프로젝트는 **브라우저 내부 2D 작업대**를 구현하는 것을 목표로 하며, 실제 공간 앵커/AR 헤드셋 전용 기능(WebXR 기반 공간 고정 등)은 범위에서 제외합니다.

## 입력 전제

- 카메라 입력: `getUserMedia()` 기반
- 보안 문맥 필요: HTTPS(또는 localhost)
- 사용자 권한 승인 필요
- 손 추적: MediaPipe Hands(Web)

## MVP 필수 기능

1. 카메라 시작 및 상태 표시
2. 손가락 방향 커서 + 흔들림 보정 + 화면 경계 제한
3. 집기(엄지-검지) 클릭
4. 길게 누르기(기본 0.6~0.8초)
5. 집은 상태 이동 기반 드래그
6. 집기 스크롤 + 관성 정지
7. 다중 창 작업대(메인 1 + 보조 2 이상)
8. 창 이동(타이틀바)
9. 창 크기 조절(모서리)
10. 홈 화면(최근 창/새 창/설정)
11. 빠른 설정(카메라/재시작/민감도/커서속도/반전/전체화면)
12. 추적 재설정(기준 자세 재정렬 + 손 미검출 복구)

## 2차(권장) 기능

- 양손 확대/축소
- 손바닥 원형 메뉴
- 배치 저장/복원
- 고정 슬롯(좌상단/우측/하단)
- 음성 보조 명령
- 얼굴 기반 추적 안내

## 기술 원칙

- 핵심 UX는 단일 손 제스처(클릭/롱프레스/드래그/스크롤) 안정화에 우선순위를 둡니다.
- 고급 제스처(양손, 손바닥 메뉴)는 오인식률과 피로도를 고려해 후순위로 둡니다.
- 카메라 거부 또는 추적 불안정 시 자동으로 마우스 모드로 폴백합니다.

자세한 설계는 `docs/architecture.md`를 참고하세요.
